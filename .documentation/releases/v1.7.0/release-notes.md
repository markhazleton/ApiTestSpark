## Highlights

v1.7.0 expands the remote API workflow from spec discovery into request execution. Server-configured remote profiles can now route endpoint calls through the host application with `EnableRemoteCallProxy`, which avoids browser CORS requirements for those profiles while keeping server-held credentials out of the browser.

This release also makes identity-aware request templating first-class. The config payload now exposes resolved `userName`, `userEmail`, and `userId` values, and configured host and remote headers can expand `{user-name}`, `{user-email}`, and `{user-id}` alongside the existing request/session GUID tokens.

## New Features

### Server-side Remote Call Proxy

`ApiTestSparkOptions.EnableRemoteCallProxy` adds an opt-in remote endpoint proxy at `/api-test-spark/remote-call`. Server-configured remote API profiles can now execute endpoint calls through the host app instead of relying on the remote API to allow browser-origin requests.

### Identity-Aware Header Templates

`/api-test-spark/config` now includes resolved user identity metadata, and configured headers can personalize outbound requests with `{user-name}`, `{user-email}`, and `{user-id}` tokens.

## Changes

### Remote Profile Management

Program.cs profiles now remain authoritative. Browser customization creates separate local copies instead of overriding the proxied server profile entry, which keeps the server proxy path stable and avoids stale local shadowing.

### Endpoint Validation and Build Hygiene

Host and remote endpoint testers now validate required path parameters before sending requests. The frontend build also suppresses the known Application Insights pure-annotation warning so release output stays focused on actionable issues.

## Bug Fixes

- **Remote explorer CORS regression**: fixed a profile-shadowing path where stale browser profiles could force browser-direct OpenAPI fetches for server-configured profiles.
- **Malformed remote requests**: fixed missing path-parameter handling so the request builders stop before sending incomplete URLs.

## Upgrade Guide

Upgrade as usual:

```bash
dotnet add package ApiTestSpark --version 1.7.0
```

To proxy server-configured remote endpoint calls through the host app, enable the new option explicitly:

```csharp
app.MapApiTestSpark(options =>
{
    options.EnableRemoteCallProxy = true;
});
```
