## Highlights

v1.1.0 delivers a clean-install option for production teams, a hardened TypeScript type system, and a fully corrected observability layer — all validated by the first full constitution compliance audit to reach 100%.

The headline feature is `EnableDemoIntegrations = false`. Any team deploying API Test Spark in a production or shared environment can now strip the built-in JokeAPI and JSONPlaceholder demo screens with a single option, leaving a focused harness showing only their host API and the API Doc Builder.

Internally, the `ErrorCategory` type union was expanded and applied consistently across `ErrorRecord`, `ErrorResponse`, and `appInsights.ts`, eliminating an unsafe string cast that had been masking the mismatch. The `ErrorBoundary` component was corrected to use the new `'React'` category and to route exclusively through `addError` rather than calling App Insights directly — ensuring the debug panel and telemetry stay in sync. ## New Features

### `EnableDemoIntegrations` Option

New `bool` property on `ApiTestSparkOptions` (default `true`). Set to `false` to hide the built-in JokeAPI and JSONPlaceholder demo screens from the home page and disable their routes (`/joke-api`, `/json-placeholder`) entirely.

```csharp
app.MapApiTestSpark(options =>
{
    options.OpenApiUrl             = "/openapi/v1.json";
    options.Environments           = ["Development", "Staging"];
    options.EnableDemoIntegrations = false;  // clean install
});
```

When disabled, the home page shows only **Host API Explorer** and **API Doc Builder**. No sample data, no external API noise.

## Changed

### TypeScript `ErrorCategory` Type Hardened

- `ErrorCategory` union expanded: `'Network' | 'API' | 'Configuration' | 'React' | 'Unknown'`
- `'React'` added for unhandled render errors caught by `ErrorBoundary`
- `ErrorRecord.category` and `ErrorResponse.category` now typed as `ErrorCategory` (was `string`)
- Unsafe `as` cast removed from `debugStore.addError` — types flow cleanly end-to-end

### `ErrorBoundary` Observability Corrected

- `componentDidCatch` now uses `category: 'React'` (was `'Unknown'`)
- Redundant direct `trackException()` call removed — `addError()` already auto-forwards to App Insights via `trackCategorizedError()`
- Debug panel and App Insights stay in sync for all React-layer errors

### Constitution Amended to v1.1.1

Three clarifications applied via full-repo review:

- **§IV**: `createRestCaller` factory recognised as second valid client pattern alongside class-based `ApiClient` extension
- **§V**: `useHarnessConfigStore` added to canonical store registry (non-persisted, session-only)
- **§VI**: `ErrorCategory` union definition updated; App Insights integration relationship documented
