## Highlights

v1.3.0 adds full **Remote API Explorer** support to ApiTestSpark. Developers can now point the harness at any remote REST API by configuring a `RemoteOpenApiUrl` in `Program.cs`. The harness fetches and parses the remote OpenAPI document via a new server-side proxy endpoint (`GET /api-test-spark/remote-spec`) that injects authentication credentials (API key or Bearer token) server-side — keeping them off the browser network tab entirely. A SSRF guard validates the remote URL scheme before any outbound call.

On the browser side, a new `Remote API Explorer` screen (`/remote-api`) and a companion `Remote API Doc Builder` (`/remote-api-docs`) provide the same endpoint browsing and documentation-capture experience as the existing host API screens. A `Remote API Config` section on the Config page allows editing all remote settings including default headers, API key, and Bearer token — with credential fields masked by default.

This release also includes two focused UX fixes: header key entry in the Config page no longer loses focus on each keystroke (isolated via an uncontrolled `KeyInput` sub-component), and newly saved headers are applied to the next request immediately without a page reload.

## New Features

### Remote API Explorer

Browse and test endpoints from any remote OpenAPI JSON document. Configure `RemoteBaseUrl` and `RemoteOpenApiUrl` in `Program.cs` — no manual UI entry required on first load. All remote configuration is editable on the Config page and persists in `localStorage`.

**Spec**: [View archived spec](specs/002-remote-openapi-config/spec.md)

### Remote Spec Proxy

`GET /api-test-spark/remote-spec` fetches the remote OpenAPI document server-side. API key and Bearer token are injected at the proxy — credentials never appear in DevTools. Non-HTTP schemes are rejected with a `400` to prevent SSRF.

### Header Token Expansion

Header values support `{session-guid}` (one UUID per page load) and `{request-guid}` (a fresh UUID on every call), expanded at request-send time. Useful for correlation IDs and session tracking across distributed systems.

### Harness Version & Build Date on About Page

The About page now shows the real NuGet package version and assembly build date, sourced from the config endpoint. Previously showed "Unknown" in NuGet embedded mode.

## Bug Fixes

- **HeadersEditor focus loss** — typing a new header key no longer unmounts the input on every keystroke. Fixed by `KeyInput` sub-component that buffers locally and commits on blur.
- **Saved headers not applied** — newly saved remote headers are now sent in the next request without a page reload. Fixed by reading live values directly from `useRemoteConfigStore` in `useHostApi`.

## Upgrade Guide

No breaking changes. Update the NuGet package:

```bash
dotnet add package ApiTestSpark --version 1.3.0
```

To enable the Remote API Explorer, add to `Program.cs`:

```csharp
app.MapApiTestSpark(options =>
{
    options.OpenApiUrl         = "/openapi/v1.json";
    options.RemoteBaseUrl      = "https://your-remote-api.com";
    options.RemoteOpenApiUrl   = "https://your-remote-api.com/openapi.json";
    // Optional auth:
    options.RemoteOpenApiApiKeyHeader = "x-api-key";
    options.RemoteOpenApiApiKeyValue  = "your-key";
    // Optional default headers (support {session-guid} / {request-guid} tokens):
    options.RemoteDefaultHeaders["correlationId"] = "{request-guid}";
});
```
