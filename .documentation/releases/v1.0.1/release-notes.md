## Highlights

This patch release delivers three major capability additions to the React SPA
and substantial enrichment to the SampleApi demo, with no breaking changes to
the `MapApiTestSpark()` public API.

The headline feature is the **API Doc Builder** — a new screen at `/api-docs`
that allows developers to select any subset of endpoints, capture live curl
commands and real HTTP responses against the running API, annotate each section
with prose, and export a complete markdown document targeted at front-end
developer agents. The output includes table of contents, parameter tables,
request body schema tables, response code tables, and fenced code blocks with
actual captured requests and responses.

The **OpenAPI metadata surface** is significantly expanded: the tester now
renders `operationId` as a copyable chip, shows all documented response codes
as expandable coloured badges, displays `default` values and `nullable` hints
in schema tables, and renders the full `info.description` as proper markdown
(including tables and fenced code blocks) in the empty state. The `info.license`
and `contact.email` fields from the OpenAPI document are now surfaced in the
header.

The **SampleApi demo** is enriched with relational cross-linked seed data:
customers have full addresses and company names, products have categories and
stock quantities, and 7 orders cover all 5 customers across every order status.
Three new read endpoints expose categories and status-filtered queries.

## New Features

### API Doc Builder (`/api-docs`)

Select endpoints from the collapsible accordion, capture live HTTP responses,
annotate sections, and generate markdown documentation for front-end developer
agents. Includes exact curl commands, full JSON responses, parameter tables,
schema tables, and response code tables.

### OpenAPI Metadata Improvements

- `operationId` displayed as a copyable chip in the endpoint tester
- All documented response codes shown as coloured badges with expandable schemas
- `default` values pre-fill the JSON scaffold and shown in schema tables
- `nullable` hints shown for optional fields in schema tables
- `info.license` and `contact.email` surfaced in the HostApiScreen header
- Full markdown renderer: bold, italic, code, headings, lists, fenced blocks, tables

### SampleApi — Relational Seed Data

- `Customer`: adds `Address` record and `Company` field; 5 seeded customers with full addresses
- `Product`: adds `Category`, `Description`, `StockQuantity`; 10 seeded products across 3 categories
- `OrderCache`: 7 seeded orders covering all 5 customers and all 5 order statuses
- New endpoints: `GET /products/categories`, `GET /products/category/{cat}`, `GET /orders/status/{status}`
- Endpoint tags changed to `"Products: Catalog"` / `"Customers: Accounts"` / `"Orders: Lifecycle"` for accordion grouping

## Bug Fixes / Dependency Fixes

- `Microsoft.OpenApi` upgraded from `2.0.0` to `2.7.6` (latest non-deprecated 2.x; 3.x breaks the AspNetCore.OpenApi source generator)
- `Microsoft.SourceLink.GitHub` updated to `10.0.300`

## Breaking Changes

None. The `MapApiTestSpark()` public API surface is unchanged.
