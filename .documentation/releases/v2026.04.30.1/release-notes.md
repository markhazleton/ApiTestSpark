## Highlights

This release transforms API Test Spark from a standalone React SPA into a dual-artifact repository: the original SPA continues to deploy to Azure Static Web Apps unchanged, and a new NuGet package (`ApiTestSpark`) embeds the SPA into any .NET 10 Minimal API project with a single call to `app.MapApiTestSpark()`.

The package autodiscovers the host app's API endpoints via its OpenAPI v3 document and renders them alongside the existing JokeAPI and JsonPlaceholder built-in examples. Auth scheme and custom request headers are configured server-side via `ApiTestSparkOptions` and delivered to the SPA through a lightweight config endpoint at `/api-test-spark/config`.

A full MSTest integration test suite (`ApiTestSpark.Tests`) guards the embedded asset pipeline, config endpoint contract, and SPA fallback behaviour in CI.

## New Features

### Portable NuGet Harness (`001-nuget-portable-harness`)

Package the compiled React SPA as embedded resources inside a .NET class library. Exposes `MapApiTestSpark()` on `IEndpointRouteBuilder` for one-line installation in any .NET 10 Minimal API project.

**Spec**: [View archived spec](specs/001-nuget-portable-harness/spec.md)

Key capabilities delivered:

- `GET /api-test-spark/` — serves the embedded SPA HTML
- `GET /api-test-spark/config` — JSON bridge: OpenAPI URL, auth scheme, default headers, base URL
- OpenAPI v3 autodiscovery: endpoints grouped by tag, deprecated badge, operation summary
- Full `$ref` resolution including .NET 10 nullable `oneOf: [null, $ref]` wrapper
- `buildJsonScaffold()` — typed JSON stub pre-fills request body from resolved schema
- Smart response rendering: array → sortable table, object → editable form, other → raw
- Environment gating, CORS without host `AddCors()`, Content-Security-Policy headers
- Dual-build via `VITE_BASE_PATH`: standalone SWA build and NuGet embedded build from same source
- MSTest integration tests with `WebApplicationFactory`
