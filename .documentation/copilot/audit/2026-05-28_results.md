## Recommendations

### High Priority (This Sprint)

1. **OBS1** — `src/utils/storage.ts`: Replace 8 `console.warn`/`console.error` calls with `useDebugStore.addError` calls using category `'Configuration'`. This is the highest-signal violation because `storage.ts` is a utility layer that the constitution explicitly covers ("all observability flows through the debug store"). The fix is mechanical: import `useDebugStore` and swap each console call.

### Medium Priority (Next Sprint)

1. **OBS2** — `src/components/Footer.tsx:19,30`: Remove `console.warn` for missing build info. Either silently return null or route through `useDebugStore`. Suggested fix: catch silently — missing build info is non-critical and should not produce any output.

2. **OBS3** — `src/components/VersionMismatchBanner.tsx:47,52`: Replace `console.warn`/`console.error` with `useDebugStore.addError('Configuration', ...)`.

3. **QUAL1** — `src/store/authStore.ts:108,153`: The `console.error` calls in catch blocks for audit log failures are technically permitted by the constitution ("MAY only appear inside catch blocks for unrecoverable errors"). However, audit log failures are *recoverable* — the auth operation succeeded and the catch is silencing the secondary telemetry failure. Consider dropping these console calls entirely or routing through the debug store.

### Low Priority (Backlog)

1. **VER1** — Add a root `CHANGELOG.md` to the repository so `/devspark.upgrade` and future audits can determine whether DevSpark is current.

2. Consider adding the `src/utils/storage.ts` unit tests noted in Principle VII (SHOULD) as the project matures — the FIFO storage path is well-isolated and testable.
