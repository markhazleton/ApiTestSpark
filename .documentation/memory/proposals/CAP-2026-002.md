## Proposed Change

### Principle VII — Testing Stance (proposed replacement)

Replace the entire principle with:

---

**React SPA (`src/`)** operates without a test framework by deliberate design.

- The React SPA quality gates are lint → typecheck → build only — no JavaScript/TypeScript test runner is in the CI pipeline (MUST-NOT change without amendment)
- The `package.json` test script intentionally echoes a no-test message — MUST NOT be silently changed (MUST)
- Unit tests SHOULD be added for critical React utility functions (e.g., `src/utils/storage.ts`, `src/utils/openApiParser.ts`) as the project matures (SHOULD)
- If a JavaScript/TypeScript test framework is introduced, it MUST be added via a constitution amendment (MUST)

**Rationale (React SPA)**: API Test Spark is a lightweight developer tool where rapid iteration and build-time safety (TypeScript strict + ESLint) provide sufficient quality assurance for the current scope. Introducing a JS/TS test framework is a significant complexity trade-off that requires explicit team agreement.

---

**.NET Library (`ApiTestSpark/`) and Demo API (`SampleApi/`)** use MSTest for integration testing.

- The .NET library MUST maintain a passing integration test suite in `ApiTestSpark.Tests/` (MUST)
- `dotnet test ApiTestSpark.Tests` MUST pass as a CI quality gate (MUST)
- New public behaviour added to `ApiTestSparkExtensions.cs` or `ApiTestSparkOptions` SHOULD have corresponding integration test coverage (SHOULD)
- MSTest is the mandated test framework for the .NET layer — no other .NET test framework may be added without amendment (MUST)

**Rationale (.NET library)**: The NuGet package ships compiled code to consumers who cannot inspect it at runtime. Integration tests using `WebApplicationFactory` / `UseTestServer` provide the only automated safety net for middleware behaviour, config serialisation, HTTP routing, and security headers. The 20 existing tests cover the full public contract surface.

---

**Combined CI quality gates (all must pass before merge):**

1. `npm run lint` — zero ESLint errors (React SPA)
2. `npm run verify` — `tsc -b` + `vite build` with zero errors (React SPA)
3. `dotnet build ApiTestSpark` — zero C# errors (.NET library)
4. `dotnet test ApiTestSpark.Tests` — all integration tests pass (.NET library)

**Rationale**: The amendment does not weaken the React SPA no-test-runner stance; it corrects a factual inaccuracy and elevates the existing .NET test suite to a first-class constitutional requirement.
