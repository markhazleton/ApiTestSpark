# ADR-001: Embedded Resource SPA Serving via EmbeddedFileProvider

## Status

Accepted

## Context

The API Test Spark React SPA needed to be distributed as a NuGet package so .NET Minimal API developers could install it with a single command. The SPA produces static assets (HTML, JS, CSS, fonts, icons) that must be served at runtime without requiring consumers to copy files into their project.

## Decision

Use `EmbeddedResource` in the `.csproj` to bake all `build/**` SPA assets into the assembly, then serve them via `EmbeddedFileProvider` from `Microsoft.Extensions.FileProviders.Embedded` wired into `UseStaticFiles`. Resource names follow the pattern `ApiTestSpark.build.{path}`.

## Consequences

### Positive

- Self-contained: no file copy step for consumers; the package is fully offline-capable
- Follows the established .NET pattern used by Scalar, Swashbuckle, and Razor Pages
- `StaticFileMiddleware` handles caching, MIME types, and range requests automatically
- An MSBuild `ValidateSpaAssets` target catches empty `build/` at pack time rather than at runtime

### Negative

- Resource name prefix must exactly match `<RootNamespace>.build.` — mismatches are silent runtime failures unless guarded by the MSBuild target and the startup assertion in `MapApiTestSpark()`
