# ADR-003: Config Endpoint as Minimal API MapGet

## Status

Accepted

## Context

The embedded SPA needs to discover the host app's OpenAPI URL, auth scheme, and default headers at startup. This information must flow from the .NET host into the React SPA without requiring the developer to hand-edit the SPA's JavaScript. A server-side endpoint is required.

## Decision

Expose `GET /api-test-spark/config` as a Minimal API endpoint (`app.MapGet`) returning `application/json`. The response shape is `{ baseUrl, openApiUrl, authScheme, defaultHeaders }`. The endpoint returns only safe metadata — never actual credential values. CORS is controlled via `ApiTestSparkOptions.CorsOrigins`.

## Consequences

### Positive

- Participates naturally in ASP.NET Core's routing, logging, and middleware pipeline
- Simpler than a bespoke middleware for a single JSON response
- `MapGet` makes the endpoint discoverable in the host app's own OpenAPI document if desired
- CORS is scoped to the config endpoint only, avoiding unintended CORS policy leakage to other routes

### Negative

- The config endpoint appears in the host app's `app.Urls` routing table, which may surface in OpenAPI docs; consumers should exclude it if unwanted
