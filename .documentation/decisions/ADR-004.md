# ADR-004: Lightweight Bespoke OpenAPI v3 Parser

## Status

Accepted

## Context

The React SPA needs to parse an OpenAPI v3 document at runtime to display the host app's endpoints. Third-party OpenAPI parser libraries (e.g., `@readme/openapi-parser`) provide full validation, `$ref` resolution, and schema dereferencing but add 200–500 KB to the bundle.

## Decision

Write a targeted `openApiParser.ts` utility that maps `paths` + `components/schemas` to `DiscoveredEndpoint[]`. The parser handles `$ref` resolution for schemas (including the .NET 10 nullable `oneOf: [null, $ref]` wrapper pattern), returns an empty array on any error rather than throwing, and logs warnings through `useDebugStore`. OpenAPI v2 (Swagger 2.0) is explicitly unsupported.

## Consequences

### Positive

- Parser is under 5 KB, keeping the NuGet package well under the 2 MB size constraint (SC-006)
- No additional runtime dependency — reduces supply-chain risk and audit surface
- Error handling is defensive: unknown `$ref`, missing `paths`, and parse failures all yield graceful degradation rather than SPA crashes

### Negative

- Does not validate the OpenAPI document against the full spec; malformed or non-standard documents may produce incomplete endpoint lists rather than clear error messages
- OpenAPI v2 documents silently return an empty endpoint list (rejected at the `openapi` field check)
