# ADR-008: Server-Side Proxy for Remote OpenAPI Spec Fetch

## Status

Accepted

## Context

ApiTestSpark needed to support fetching an OpenAPI JSON document from a remote server that may require authentication (API key or Bearer token). A browser-side fetch was not viable: it would expose credentials in the browser network tab, and CORS restrictions on the remote server would block the request without server changes. The feature also needed to work in the common case where a developer controls the .NET host but not the remote API server.

## Decision

A new .NET endpoint (`GET /api-test-spark/remote-spec`) acts as a server-side proxy. The React SPA requests the proxy rather than the remote URL directly. The proxy reads `RemoteOpenApiUrl` and credentials from `ApiTestSparkOptions` (startup configuration), performs the outbound HTTP request server-side, validates the response content-type, and returns the raw JSON to the browser. Credentials never appear on the browser network tab.

A dedicated `useRemoteConfigStore` (Zustand persist, key `api-test-spark-remote-config`) holds the live remote configuration in the React layer. `useHostApi` reads from this store directly so credential and header updates saved on the Config page take effect on the next request without a page reload.

## Consequences

### Positive

- Credentials are never visible in the browser network tab or DevTools — the primary security goal is achieved.
- CORS issues with the remote server are eliminated; the proxy makes a same-origin call from the .NET host's perspective.
- The CSP `connect-src` directive is automatically extended with `RemoteBaseUrl`, so direct endpoint calls (path 2) are browser-permitted without modifying the remote server.
- An SSRF guard (scheme validation: `http://`/`https://` only) is enforced at the proxy before any outbound call.

### Negative

- UI-entered or UI-edited credential values do not reach the proxy until the .NET host is restarted with updated `Program.cs` values. This is a known limitation documented in the UI near the credential fields; resolving it would require passing credentials from the browser to the proxy, defeating the security design.
- The proxy depends on `ApiTestSparkOptions` being set at startup — dynamic credential updates require an application restart.
