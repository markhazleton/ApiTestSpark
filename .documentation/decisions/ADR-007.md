# ADR-007: Native `<details>`/`<summary>` for Collapsible Nested Object Sections

## Status

Accepted

## Context

The response renderer needed a collapsible widget to reveal editable sub-forms for depth-1 nested objects. The widget must be collapsed by default, keyboard accessible, and introduce no new dependencies.

## Decision

Use native HTML `<details>`/`<summary>` elements, matching the `LazyDetails` pattern already present in `DebugPanelUtils.tsx`. Collapsed by default via absence of the `open` attribute (FR-012).

A custom React collapse component was rejected (adds complexity and a potential new dependency). A persistent visible expanded state was rejected (spec FR-012 explicitly requires collapsed-by-default).

## Consequences

### Positive

- Zero JavaScript overhead — browser-native toggle behaviour
- Keyboard accessible and screen-reader friendly by default
- Consistent with the existing `LazyDetails` pattern already in the codebase

### Negative

- CSS styling of `<details>` disclosure triangles varies by browser; custom visual treatment requires additional CSS
